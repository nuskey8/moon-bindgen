# moon-bindgen

Automatically generates MoonBit FFI bindings to Rust/C libraries.

## Overview

`moon-bindgen` is a library for generating MoonBit FFI bindings from Rust or C libraries. It parses Rust FFI code and automatically generates MoonBit FFI code directly from `build.rs`. When used in combination with [rust-bindgen](https://crates.io/crates/bindgen), it can also generate FFI bindings for C libraries.

The concept of `moon-bindgen` is inspired by [csbindgen](https://github.com/Cysharp/csbindgen).

## Usage

Create a new package with `cargo new` and add `moon_bindgen` to your `[build-dependencies]`.

```bash
cargo add --build moon_bindgen

```

```toml
[package]
name = "example"
version = "0.1.0"

[lib]
crate-type = ["cdylib"]

[build-dependencies]
moon_bindgen = "1.0.0"

# optional (C FFI)
bindgen = "0.72" 
cc = "1.2.67"

```

## Rust to MoonBit

`moon-bindgen` can generate MoonBit FFI code directly from Rust FFI code.

```rust
// lib.rs

#[unsafe(no_mangle)]
pub extern "C" fn my_add(a: i32, b: i32) -> i32 {
    a + b
}

```

Add the following code to your `build.rs`:

```rust
use std::error::Error;

fn main() -> Result<(), Box<dyn Error>> {
    moon_bindgen::Builder::default()
        .input_extern_file("src/lib.rs")
        .generate()?
        .write_moonbit_ffi_to_file("../ffi.mbt")?;
}

```

Running `cargo build` will generate the following MoonBit code:

```mbt
// ffi.mbt
// Generated by moon-bindgen. Do not edit.

extern "c" fn my_add(a : Int, b : Int) -> Int = "my_add"

```

> [!NOTE]
> When using built libraries such as `.dll`, `.dylib`, or `.so`, you need to configure `moon.pkg` so that the linker can link them properly.

## C to MoonBit

By using `moon-bindgen` together with `rust-bindgen`, you can generate MoonBit FFI bindings from C header files.

```h
// add.h
int add(int a, int b);

```

```rust
use std::error::Error;

fn main() -> Result<(), Box<dyn Error>> {
   bindgen::Builder::default()
        .header("includes/add.h")
        .generate()?
        .write_to_file("src/ffi.rs")?;

    let bindings = moon_bindgen::Builder::default()
        .input_bindgen_file("src/ffi.rs")
        .generate()?;
    bindings.write_to_file("../ffi.mbt", "../ffi_stub.c")?;

    Ok(())
}

```

## Ownership

When passing resources via FFI in MoonBit, you need to configure the ownership of arguments using either `#borrow` or `#owned`. By default, all arguments are treated as `#borrow`, but you can specify the ownership for each function argument using `moonbit_ownership_resolver()`.

```rust
moon_bindgen::Builder::default()
    .input_bindgen_file("src/ffi.rs")
    .moonbit_ownership_resolver(|func_name, param_name| {
        moon_bindgen::Ownership::Borrowed
    })
    .generate()?
    .write_moonbit_ffi_to_file("../ffi.mbt")?;

```

## Nullability

You can modify pointer nullability using `moonbit_nullability_resolver()`. By default, only return values ​​are treated as nullable.

```rust
moon_bindgen::Builder::default()
    .input_bindgen_file("src/ffi.rs")
    .moonbit_nullability_resolver(|function, position| match (function, position) {
        ("context_get", moon_bindgen::NullabilityPosition::Return) => {
            moon_bindgen::Nullability::NonNull
        }
        ("context_set", moon_bindgen::NullabilityPosition::Parameter(name))
            if name == "context" => moon_bindgen::Nullability::Nullable,
        _ => moon_bindgen::Nullability::Unspecified,
    })
    .generate()?;
```

## Type Marshalling

| Rust                                 | MoonBit                    |
| ------------------------------------ | -------------------------- |
| `()`                                 | `Unit`                     |
| `i8`, `u8`                           | `Byte`                     |
| `i16`, `u16`, `i32`                  | `Int`                      |
| `u32`                                | `UInt`                     |
| `i64`, `isize`                       | `Int64`                    |
| `u64`, `usize`                       | `UInt64`                   |
| `f32`                                | `Float`                    |
| `f64`                                | `Double`                   |
| `bool`                               | `Bool`                     |
| `c_char`, `c_schar`, `c_uchar`       | `Byte`                     |
| `c_short`, `c_ushort`, `c_int`       | `Int`                      |
| `c_uint`                             | `UInt`                     |
| `c_long`, `c_longlong`               | `Int64`                    |
| `c_ulong`, `c_ulonglong`, `size_t`   | `UInt64`                   |
| `c_float`                            | `Float`                    |
| `c_double`                           | `Double`                   |
| `c_void`                             | `Unit`                     |
| `*const c_char`, `*mut c_char`       | `Bytes`                    |
| `*const i8`, `*mut i8`               | `Bytes`                    |
| `*const u8`, `*mut u8`               | `Bytes`                    |
| `*const c_void`, `*mut c_void`       | `RawPtr`                   |
| `*const T`, `*mut T`                 | `Ref[T]`, `#external` type |
| `extern "C" fn(...)`                 | `FuncRef`                  |
| `Option<extern "C" fn(...)>`         | `FuncRef?`                 |
| Passing `#[repr(C)] struct` by value | MoonBit `struct` + C stub  |

### C Stub

MoonBit's standard structs and C ABI structs do not share the same memory layout. For functions that accept or return a `#[repr(C)] struct` by value, `moon-bindgen` generates a C stub for ABI conversion.

```rust
let bindings = moon_bindgen::Builder::default()
    .input_bindgen_file("src/ffi.rs")
    .moonbit_visibility(moon_bindgen::Visibility::Public)
    .c_stub_file_header("#include \"library.h\"")
    .c_stub_file_footer("// end of generated adapters")
    .generate()?;

bindings.write_to_file("../ffi.mbt", "../ffi_stub.c")?;

```

The generated stub should be added to the `native-stub` field in your `moon.pkg`.

```moonbit
options(
  "native-stub": ["ffi_stub.c"],
  targets: { "ffi.mbt": ["native"] },
)

```

## Builder Options

```rust
use heck::{ToShoutySnakeCase, ToSnakeCase, ToUpperCamelCase};
use std::error::Error;

fn main() -> Result<(), Box<dyn Error>> {
    let bindings = moon_bindgen::Builder::default()
        .input_bindgen_file("src/c_bindings.rs")
        .input_extern_file("src/lib.rs")
        .moonbit_file_header("")
        .moonbit_file_footer("")
        .c_stub_file_header("#include \"library.h\"")
        .c_stub_file_footer("// end of generated adapters")
        .moonbit_visibility(moon_bindgen::Visibility::Public)
        .function_filter(|name| !name.starts_with("_"))
        .type_filter(|name| !name.starts_with("_"))
        .constant_filter(|name| !name.starts_with("_"))
        .moonbit_function_rename(|name| name.to_snake_case())
        .moonbit_type_rename(|name| name.to_upper_camel_case())
        .moonbit_constant_rename(|name| name.to_shouty_snake_case())
        .moonbit_ownership_resolver(|function, parameter| moon_bindgen::Ownership::Borrowed)
        .generate()?;

    bindings.write_to_file("../ffi.mbt", "../ffi_stub.c")?;

    Ok(())
}

```
